# OMM (Open Match Model)

[READ in English](https://github.com/NelikKKL/OMM/blob/main/EN.README.md)

**OMM** — это сверхлегкий формат 3D-моделей и автономный движок на чистом JavaScript. Создан для быстрой интеграции 3D-графики в веб-проекты без использования тяжелых библиотек (Zero Dependencies).

## 🚀 Быстрый старт

1. Поместите `omm-core.js` в папку с проектом.
2. Подключите скрипт в конце вашего HTML:
   ```html
   <script src="omm-core.js"></script>
   ```
3. Используйте тег модели:
   ```html
   <omm-model src="my_model.omm" autorate></omm-model>
   ```
### <omm-model>
в этот тег можно добавить так *` <omm-model src="model.omm"  сюда`

можно добавить
`autorate` - модель сама крутиться
`freer` - полностю свободное вращение, уменьшение отдаление

## 📝 Синтаксис и возможности .omm

Файл `.omm` — это текстовый документ. Каждая новая фигура начинается с ключевого слова.

### 1. Геометрия
* `cube3` — стандартный куб.
* `cube3 : 4` — плита (высота уменьшена в 4 раза). Цифра после `:` определяет тонкость объекта.
* `image3` — плоская поверхность (биллборд) в 3D пространстве. Идеально для иконок или травы.

### 2. Трансформации
Параметры пишутся под фигурой или в одну строку:
* `x(n) y(n) z(n)` — позиция. В OMM `y` идет вверх (отрицательные значения поднимают объект).
* `scale(n)` — масштабирование. По умолчанию размер грани 50px.
* `rr(deg)` — поворот вправо (вокруг оси Y).
* `ru(deg)` — поворот вверх (вокруг оси X).

### 3. Анимации
Вы можете указать несколько точек
пример
* `animation(x0 y0 z0, x0 y200 z0)`
+ вы можете указать только одно нужное значение в анимации
* `animation(y200 , y0)`
можно использывать rr и другое

### 4. Визуализация и Текстуры
* `color(R, G, B)` — цвет заливки (например, `color(0, 255, 100)`).
* **Текстуры**: Команда `texture(...)` заменяет цвет изображением.
    * **Внешняя ссылка**: `texture(https://site.com/img.png)`
    * **Локальная (Base64)**: В редакторе Studio при загрузке картинки она автоматически кодируется:
        `texture(data:image/png;base64,iVBOR...)`
    * *Примечание:* Текстура накладывается на все грани фигуры автоматически.

## 💡 Продвинутое использование (API)

Вы можете управлять моделью прямо из HTML через атрибуты тега `<omm-model>`:

| Атрибут | Описание |
| :--- | :--- |
| `src` | Путь к файлу `.omm`. |
| `autorate` | Если присутствует, модель будет бесконечно вращаться. |
| `textContent` | Можно писать код модели прямо внутри тега. |

**Пример встроенного кода:**
```html
<omm-model>
  cube3
  color(255, 0, 0)
  scale(2)
</omm-model>
```

## ⚠️ Технические нюансы
* **CORS**: При работе с локальными файлами (`file://`) браузер может блокировать загрузку `.omm`. Рекомендуется использовать локальный сервер (Live Server VS Code, Node.js, python).
* **Base64**: Использование Base64 внутри `.omm` делает файл тяжелее, но позволяет модели работать автономно без внешних картинок
